# 配置文件升级计划

## 评估结论：建议基本合理，需要修复代码错误并调整实施细节

---

## 一、建议的优点

### 1. 配置更丰富
- 新增 `minParams`, `maxParams` 字段，控制参数数量范围
- 新增 `id` 字段，便于 UI 组件引用
- L4 的 `complexity` 从 4 提升到 5，`steps` 从 5 提升到 6，更符合压轴题特征

### 2. 变量旋钮池设计优秀
- `VARIABLE_KNOBS_POOL` 提供具体的参数值（整数、分数、根式、字母、混合）
- `filterKnobsByLevel()` 根据难度等级过滤旋钮，L2 只用简单参数，L4 可用复杂参数
- 权重设计合理：基础整数权重 4，纯字母参数权重 1

### 3. 目录结构更规范
- `src/config/` 专门存放配置文件，与 `src/utils/` 工具函数分离

## 二、需要修复的问题

### 1. 代码语法错误

**difficultyConfig.js 中的错误：**
```javascript
// 错误代码
const tier = DIFFICULTY_TIERS.find(t => 
  score >= t.eloRange[0] && score  // 不完整！缺少比较条件

// 正确代码
const tier = DIFFICULTY_TIERS.find(t => 
  score >= t.eloRange[0] && score <= t.eloRange[1]
);
```

**getAllTiers() 中的模板字符串错误：**
```javascript
// 错误代码
range: {t.eloRange[0]}-{t.eloRange[1] === Infinity ? '+' : t.eloRange[1]}

// 正确代码
range: `${t.eloRange[0]}-${t.eloRange[1] === Infinity ? '+' : t.eloRange[1]}`
```

### 2. 重复定义问题
当前 `problemLogic.js` 中已有 `getDifficultyByElo` 函数，需要决定：
- 方案 A：删除 `problemLogic.js` 中的硬编码版本，改为从 config 导入
- 方案 B：保留 `problemLogic.js` 作为门面，内部调用 config

**推荐方案 A**：彻底消除硬编码
