# 杀手锏分级规划方案

## 问题诊断

### 原匹配脚本的问题

**之前的匹配结果有误！** M04 被分配了约 20 个杀手锏，但其中大部分是其他模块的：

| 错误匹配                                                       | 应属于      |
| ---------------------------------------------------------- | -------- |
| S-FUNC-02, S-FUNC-04, S-FUNC-05, S-FUNC-06, S-FUNC-08      | M03（函数）  |
| S-DERIV-03, S-DERIV-04, S-DERIV-09, S-DERIV-10, S-DERIV-11 | M11（导数）  |
| S-INEQ-02, S-INEQ-05, S-INEQ-06, S-INEQ-07, S-INEQ-10      | M02（不等式） |
| S-SEQ-02, S-SEQ-04, S-SEQ-08, S-SEQ-09, S-SEQ-10           | M08（数列）  |

**正确情况：**

* M04（指对数）应该只有 **2 个**杀手锏：S-LOG-02, S-LOG-05

* M05（向量）应该只有 **3 个**杀手锏：S-VEC-02, S-VEC-04, S-VEC-05

## 六、实施步骤

1. **移除 Level 1 杀手锏**（8 个）从 `weapon_details.json`
2. **重写匹配脚本**：严格按母题分配杀手锏
3. **重新匹配 M04/M05**：只匹配本模块的杀手锏
4. **验证结果**：确保每个母题的杀手锏数量正确

***

## 七、总结

**问题根源**：之前的匹配脚本允许跨模块匹配，导致 M04 被分配了大量其他模块的杀手锏。

**解决方案**：严格按母题分配，M04 只能用 S-LOG-xx，M05 只能用 S-VEC-xx。

**最终杀手锏数量**：

* 移除 Level 1：8 个

* 保留 Level 2+：**39 个**

* 覆盖母题：12 个（M01-M12）
