# 双难度双问生成器 + 相对复杂度排序 完整实施方案

## 数据结构确认

### 变量因子与变例的关系

**确认：变量因子（variable\_knobs）是和变例（variation）绑定的！**

```
M01.json
├── specialties (专项列表)
│   └── spec_id: "V1" (集合的运算与关系)
│       └── variations (变例列表)
│           └── var_id: "1.1" (基本运算与要素识别)
│               ├── variable_knobs (变量因子池) ← 绑定在变例上！
│               │   ├── element_type: [...]
│               │   ├── operation_type: [...]
│               │   └── trap_type: [...]
│               ├── master_benchmarks (标杆题) ← 多道题共享同一套因子
│               └── original_pool (原始题目池) ← 多道题共享同一套因子
```

**关键理解**：

* 一个变例 = 一套变量因子池

* 一个变例可以有多道标杆题（不同难度）

* 这些标杆题共享同一套变量因子

* **相对复杂度排序是在同一套因子池内选择不同复杂度的组合**

***

## 方案整合

将两个优化方案整合为一个完整的出题逻辑升级方案：

### 方案一：双难度双问生成器

* **第(1)问**：levelBase（错题难度/用户选择难度）→ 解决"会不会"

* **第(2)问**：levelTarget（ELO能力难度）→ 解决"熟不熟、深不深"

### 方案二：相对复杂度排序

* **L2**：从因子池中挑选【复杂度最低】的部分

* **L3**：从因子池中挑选【中等复杂度】的部分

* **L4**：从因子池中挑选【复杂度最高】的部分

***

### 3. 相对复杂度排序策略

```javascript
const getSelectionStrategy = (level) => {
  switch(level) {
    case 'L2': 
      return "从因子池中挑选【复杂度最低】的那一部分数值/参数（最基础、最直观的配置）";
    case 'L3': 
      return "从因子池中挑选【中等复杂度】的数值/参数（具有一定计算量或逻辑转折）";
    case 'L4': 
      return "从因子池中挑选【复杂度最高】的那一部分数值/参数（最繁琐、最隐蔽或最具综合性的配置）";
    default: 
      return "从因子池中随机挑选";
  }
};
```

