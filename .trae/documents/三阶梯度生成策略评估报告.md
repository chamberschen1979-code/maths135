# 三阶梯度生成策略评估报告

## 建议概述

用户建议将出题逻辑从"随机生成3道同难度题"升级为"三阶梯度生成"：
- Q1（基础巩固型）：流畅复刻，数值友好化
- Q2（变式进阶型）：结构变式，数值复杂化
- Q3（极限施压型）：条件隐蔽，参数动态化

## 合理性分析

### ✅ 优点

1. **教育价值高**
   - 符合认知科学的"最近发展区"理论
   - Q1→Q2→Q3 的难度递进设计合理
   - 避免"三道题像一道题"的无效训练

2. **维度扰动模型科学**
   - 数值脏度：整数→分数/根号/无理数
   - 结构隐蔽：显性条件→隐性条件
   - 参数动态：定值→含参讨论
   - 几何位置：特殊位置→一般位置
   - 逻辑链条：单步推导→多步嵌套

3. **区分度明确**
   - Q1：建信心、通流程
   - Q2：练熟练度、防粗心
   - Q3：抗压训练、破局能力

### ⚠️ 潜在问题

1. **与现有 Elo 系统冲突**
   - 现有系统：根据 Elo 分数决定难度（L2/L3/L4）
   - 建议方案：固定三阶梯度（Q1/Q2/Q3）
   - **冲突点**：Elo < 1800 的学员生成 3 道 L2，如何应用三阶梯度？

2. **复杂度增加**
   - Prompt 复杂度大幅增加
   - AI 可能难以准确理解"数值脏化"、"条件隐蔽"等抽象概念
   - 需要重构 `handleGenerateBundle` 函数

3. **AI 输出质量不稳定**
   - "极限施压版"的生成依赖 AI 的深度推理能力
   - 可能出现 Q3 难度不够或偏题的情况

## 兼容性评估

### 现有系统架构

```
Elo < 1800: 3道 L2（同难度）
Elo 1800-2500: 1道 L2 + 2道 L3（跨难度）
Elo >= 2501: 1道 L2/L3 + 1道 L4 + 1道跨专项
```

### 建议方案架构

```
每个母题: Q1(基础) + Q2(进阶) + Q3(施压)
```

### 融合方案

**推荐：将三阶梯度作为"同一难度内的变式策略"**

```
Elo < 1800: 
  - Q1: L2 基础版（数值友好）
  - Q2: L2 进阶版（数值复杂）
  - Q3: L2 压力版（结构隐蔽）

Elo 1800-2500:
  - Q1: L2 回顾版
  - Q2: L3 进阶版
  - Q3: L3 压力版

Elo >= 2501:
  - Q1: L3 回顾版
  - Q2: L4 挑战版
  - Q3: L4 压力版
```

## 实施建议

### 第一阶段：Prompt 增强（低成本）

在现有 System Prompt 中增加"变式维度"约束：

```
【变式约束】
生成题目时，请从以下维度制造差异：
1. 数值脏度：整数→分数/根号/无理数
2. 结构隐蔽：显性条件→隐性条件
3. 参数动态：定值→含参讨论
4. 逻辑链条：单步推导→多步嵌套
```

### 第二阶段：三阶梯度指令（中成本）

修改 `handleGenerateBundle` 函数，为每道题注入不同的 `difficulty_profile`：

```javascript
const profiles = [
  { role: "基础", instruction: "数值友好化，移除旁支干扰", difficulty_delta: -0.1 },
  { role: "进阶", instruction: "数值复杂化，改变提问角度", difficulty_delta: +0.1 },
  { role: "施压", instruction: "引入参数讨论，隐藏关键条件", difficulty_delta: +0.3 }
]
```

### 第三阶段：完整重构（高成本）

完全重构出题逻辑，实现"三阶梯度生成策略"的完整实施。

## 结论

**建议合理，但需要分阶段实施：**

1. **立即实施**：在 Prompt 中增加"变式维度"约束（低成本，立竿见影）
2. **短期实施**：为每道题注入不同的 `difficulty_profile`（中成本，效果显著）
3. **长期规划**：完整重构出题逻辑（高成本，需充分测试）

**核心改进点**：
- 不是"替代"现有 Elo 系统，而是"增强"同一难度内的变式质量
- 将"三阶梯度"作为"变式策略"，而非"难度分级"
