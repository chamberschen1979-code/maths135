## 二、建议代码的问题

### 1. 数据结构不一致

**建议代码**：
```javascript

  // ...

```

**当前代码**：
```javascript
export const buildCrossFileIndex = (motifDataMap) => {

  // ...

};
```

**问题**：如果改用 Map，需要同时修改 `findMotifData` 的调用方式：
- 当前：`crossFileIndex[variant]`
- 建议：`crossIndex.has(targetId)` / `crossIndex.get(targetId)`

### 2. selectBenchmark 实现差异

**建议代码**：从 `motifData.benchmarks` 查找
**当前代码**：从 `motifData.specialties[].variations[].master_benchmarks` 查找

当前实现更符合 M 文件的实际结构。

### 3. findMotifData 签名不兼容

**建议代码**：
```javascript
export const findMotifData = async (targetId, crossIndex, loaderFunc) => {
  if (crossIndex.has(targetId)) {  // 假设 Map
    return crossIndex.get(targetId);
  }
}
```

**当前代码**：
```javascript
export const findMotifData = async (targetId, crossFileIndex, loadMotifDataFn = null) => {
  // 支持 ID 变体（下划线/连字符/大小写）
  const idVariants = [targetId, normalizeId(targetId), ...]
  for (const variant of idVariants) {
    if (crossFileIndex[variant] && crossFileIndex[variant].length > 0) {
      return crossFileIndex[variant][0]
    }
  }
}
```

当前实现更健壮，支持 ID 变体匹配。
