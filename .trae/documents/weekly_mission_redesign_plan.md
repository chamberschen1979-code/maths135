重新规划周度任务页面，将原有的"激活母题"、"总积分"等概念移除，改为以"出题范围"为核心的卡片式布局。

## 三、核心功能模块

### 3.1 错题区模块

#### 3.1.1 导入错题
- **功能**：从外部导入错题（如拍照、PDF等）
- **逻辑**：
  - 识别错题所属的母题、专项、变例
  - 只改变该变例的等级状态灯：
    - 如果当前是红色 → 保持红色
    - 如果当前是绿色 → 改为红色
  - **不扣除 Elo 积分**
- **UI**：
  - 按钮：`[导入错题]`
  - 点击后打开导入对话框（支持图片上传、手动选择母题）

#### 3.1.2 训练错题
- **功能**：展示来自"每周任务"和"每日训练"中做错的题目
- **逻辑**：
  - 从 `errorNotebook` 中筛选 `source: 'training'` 或 `source: 'weekly'` 的错题
  - 按母题分组展示
- **UI**：
  - 按钮：`[训练错题]`
  - 展开显示错题列表

#### 3.1.3 错题母题卡片
- 展示所有错题对应的母题
- 卡片显示：母题名称、错题数量、状态灯

### 3.2 自选区模块

#### 3.2.1 母题选择器
- **功能**：在激活的母题（Elo > 1001）中自由选择
- **颗粒度**：可选择到专项级别（如 M01-V1）
- **排除逻辑**：
  - 如果母题已在错题区出现，则不可选择（置灰）
- **UI**：
  - 按钮：`[选择母题]`
  - 弹出选择器，支持多选
  - 显示已选择的母题卡片

### 3.3 强化训练区模块

#### 3.3.1 自动选择逻辑
- **功能**：自动选择 Elo 分数最低的 2 个激活母题
- **排除逻辑**：
  - 如果母题已在错题区或自选区出现，则跳过
  - 顺延选择下一个最低分母题
- **UI**：
  - 自动展示选中的母题卡片
  - 卡片显示：母题名称、Elo 分数

### 3.4 生成任务模块

#### 3.4.1 生成任务按钮
- **功能**：调用 AI 生成题目
- **逻辑**：
  - 合并错题区、自选区、强化训练区的母题
  - 为每个母题生成 1-3 道题目
  - 题目包含：题干、解析、答案
- **UI**：
  - 按钮：`[生成任务]`
  - 加载状态显示

#### 3.4.2 打印任务按钮
- **功能**：生成可打印的任务单
- **逻辑**：
  - 只包含题干和答题区（空白块）
  - **不包含解析**
- **UI**：
  - 按钮：`[打印任务]`
  - 打开打印预览页面

### 3.5 答案录入模块

#### 3.5.1 手工录入
- **功能**：手动输入答案
- **UI**：
  - 每道题目下方有答案输入框
  - 提交按钮

#### 3.5.2 扫描录入
- **功能**：拍照识别答案（类似错题诊断页面）
- **逻辑**：
  - 调用 AI 识别图片中的答案
  - 自动填充到答案框

#### 3.5.3 诊断与打分
- **功能**：AI 诊断答案并打分
- **逻辑**：
  - 对比标准答案
  - 给出分数和诊断意见
  - 更新 Elo 分数
  - 更新变例等级状态灯

## 四、数据结构设计

### 4.1 扩展 errorNotebook 结构

```typescript
interface ErrorItem {
  id: string
  targetId: string           // 母题ID
  specId?: string            // 专项ID（可选）
  varId?: string             // 变例ID（可选）
  diagnosis: string          // 诊断信息
  level: string              // 难度等级
  source: 'import' | 'training' | 'weekly'  // 来源
  addedAt: string
  resolved: boolean
  resolvedAt?: string
  score?: number             // 得分（录入后）
}
```

### 4.2 新增 weeklyPlan 结构

```typescript
interface WeeklyPlan {
  // 错题区
  errorMotifs: string[]      // 错题对应的母题ID列表
  
  // 自选区
  selectedMotifs: Array<{
    motifId: string
    specId?: string          // 专项级别选择
  }>
  
  // 强化训练区（自动计算）
  reinforcementMotifs: string[]
  
  // 生成的任务
  generatedTasks: Array<{
    id: string
    motifId: string
    specId?: string
    varId?: string
    problem: string          // 题干
    analysis: string         // 解析
    answer: string           // 答案
    userAnswer?: string      // 用户答案
    score?: number           // 得分
    status: 'pending' | 'completed' | 'reviewed'
  }>
  
  weekStart: string
  weekEnd: string
}
```

### 4.3 变例等级状态灯

```typescript
interface VariationStatus {
  motifId: string
  specId: string
  varId: string
  level: 'L2' | 'L3' | 'L4'
  status: 'red' | 'yellow' | 'green'  // 状态灯颜色
  lastUpdated: string
}
```

## 七、风险与注意事项

1. **数据迁移**：需要处理现有 `errorNotebook` 数据的兼容性
2. **性能优化**：母题选择器需要处理大量数据，考虑虚拟滚动
3. **AI 调用**：答案诊断需要额外的 AI 调用，注意 API 配额
4. **打印样式**：打印预览需要专门的 CSS 样式

## 八、验收标准

1. ✅ 错题区正确展示导入错题和训练错题
2. ✅ 自选区支持专项级别选择，正确排除已在错题区的母题
3. ✅ 强化训练区自动选择最低 Elo 母题
4. ✅ 生成任务功能正常，包含题干、解析、答案
5. ✅ 打印任务只显示题干和答题区，不显示解析
6. ✅ 答案录入支持手工和扫描两种方式
7. ✅ 诊断打分后正确更新 Elo 和状态灯
