
## 三、问题定位

### 3.1 当前逻辑的局限性

**问题**：两级路由只针对**当前选定的变例** (`targetVar`)，不会跨变例查找。

```
当前逻辑：
选定变例 → 查找 master_benchmarks → 没找到 → 查找 original_pool → 没找到 → 报错

理想逻辑：
选定变例 → 查找 master_benchmarks → 没找到 → 查找 original_pool → 没找到 → 
跨变例查找 → 找到 → 使用
```

### 3.2 为什么会出现"数据缺失"报错？

| 场景 | 原因 |
|------|------|
| 自动选择变例时选择了变例A | 变例A的 `original_pool` 没有 L4 难度的题目 |
| 但变例B、C有 L4 题目 | 代码不会去变例B、C查找 |
| 结果 | 报错"数据缺失"，但实际数据存在 |

---

## 四、修复方案

### 方案：添加第三级路由（跨变例查找）

```javascript
// 第一优先级: master_benchmarks (当前变例)
// 第二优先级: original_pool (当前变例)
// 第三优先级: 跨变例查找 original_pool

if (benchmarkQuestions.length === 0 && motifObj?.specialties) {
  for (const spec of motifObj.specialties) {
    if (benchmarkQuestions.length > 0) break
    for (const v of spec.variations || []) {
      if (v.original_pool) {
        const poolMatches = v.original_pool.filter(q => q.level === frontendLevel)
        if (poolMatches.length > 0) {
          const randomIndex = Math.floor(Math.random() * poolMatches.length)
          benchmarkQuestions = [{
            level: frontendLevel,
            problem: poolMatches[randomIndex].desc || poolMatches[randomIndex].problem || '',
            logic_key: poolMatches[randomIndex].logic_key || '高考真题/名校模考',
            analysis: poolMatches[randomIndex].analysis || null
          }]
          console.log(`【跨变例种子选择】从 ${spec.spec_name}/${v.name} 的 original_pool 选择 ${frontendLevel} 难度题目`)
          break
        }
      }
    }
  }
}
```
