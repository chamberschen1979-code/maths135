# 杀手锏出题速度与AI诊断精度优化计划

## 问题分析

### 问题一：杀手锏出题比每周任务出题慢

**原因分析：**

1. **模型差异**：
   - 杀手锏出题使用 `TEXT_MODEL_NAME = 'qwen-plus'` (文本模型)
   - 每周任务出题使用 `VISION_MODEL_NAME` (视觉模型，带图片输入)
   - 两者都调用同一个 API 端点，但杀手锏的 Prompt 更复杂

2. **Prompt 复杂度对比**：
   - **杀手锏 QUESTION_GEN_PROMPT**：约 800+ 字符，包含多个命题铁律、输出要求、重要提醒
   - **每周任务 WEEKLY_TASK_GEN_PROMPT**：约 200 字符，简单的变式题生成要求

3. **串行调用问题**：
   - 杀手锏认证需要连续生成 5 道题（虽然改为动态生成，但每题仍需等待）
   - 每周任务只需生成 1 道变式题

4. **额外处理**：
   - 杀手锏出题后有 `cleanSolution` 清洗函数处理
   - 每周任务没有额外处理

### 问题二：AI诊断精度不够

**原因分析：**

1. **DIAGNOSIS_PROMPT 问题**：
   - Prompt 过于简单，只要求返回母题ID和评语
   - 没有要求AI进行详细的步骤分析
   - 缺少对解题过程的深度检查

2. **缺少上下文**：
   - 诊断时没有提供学生的历史学习数据
   - 没有结合学生的薄弱点进行针对性诊断

3. **输出格式限制**：
   - 只返回 targetId, greenSubIds, message
   - 没有要求返回详细的错误分析和改进建议

## 优化方案

### 方案一：优化杀手锏出题速度

1. **简化 Prompt**：
   - 精简 QUESTION_GEN_PROMPT，移除冗余描述
   - 将核心要求压缩到 400 字符以内

2. **使用更快的模型**：
   - 考虑将 TEXT_MODEL_NAME 改为 `qwen-turbo` (更快的模型)
   - 或使用流式输出减少等待感

3. **并行预生成**：
   - 在 Intro 页面时预加载第一题
   - 用户阅读规则时后台已经开始生成

### 方案二：提升AI诊断精度

1. **增强 DIAGNOSIS_PROMPT**：
   - 要求AI进行详细的步骤分析
   - 增加对常见错误的识别要求
   - 要求返回具体的改进建议

2. **丰富输出内容**：
   - 增加 `stepAnalysis` 字段（步骤分析）
   - 增加 `commonErrors` 字段（常见错误）
   - 增加 `improvementPlan` 字段（改进计划）

3. **结合学生数据**：
   - 传入学生的历史错题数据
   - 结合薄弱母题进行针对性诊断

## 实施步骤

### 步骤一：优化杀手锏出题 Prompt
- 精简 QUESTION_GEN_PROMPT
- 移除冗余的"命题铁律"描述
- 保留核心的 JSON 输出要求

### 步骤二：考虑使用更快的模型
- 评估 `qwen-turbo` 模型的效果
- 或保持 `qwen-plus` 但优化 Prompt

### 步骤三：增强 AI 诊断 Prompt
- 重写 DIAGNOSIS_PROMPT
- 增加详细的步骤分析要求
- 增加常见错误识别要求
- 丰富输出格式

### 步骤四：更新相关组件
- 更新诊断结果展示组件
- 显示更详细的诊断信息

## 预期效果

1. **出题速度提升 30-50%**
2. **诊断精度提升，能识别更多细节错误**
3. **用户体验更流畅**
