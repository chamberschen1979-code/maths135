# 简化版权限管理实施计划

## 需求

1. **新用户不能自己注册**，只能管理员创建
2. **未登录用户不能进入首页**，直接跳转到登录页
3. **管理员可配置普通用户的“学情评估”权限**，只有开通后普通用户才有用学情评估的权限

### 管理员账号

* 用户名：`ccb`

* 密码：`admin2026`

***

## 设计方案

### 用户角色

| 角色        | 登录首页      | 使用学情评估 | 创建/删除用户 | 配置用户权限 |
| --------- | --------- | ------ | ------- | ------ |
| 管理员 (ccb) | ✓         | ✓      | ✓       | ✓      |
| 普通用户      | ✓（需管理员创建） | ✗（需开通） | ✗       | ✗      |

### 功能列表

#### 1. 登录页面

* 用户名 + 密码登录

* 无注册入口（注册只能管理员操作）

#### 2. 管理员面板

* 创建新用户（设置用户名、初始密码）

* 删除用户

* 查看所有用户列表及状态

* **配置每个用户的“学情评估”权限**（开通/关闭）

#### 3. 普通用户首页

* 登录后可见首页

* 如果没有“学情评估”权限，则看不到相关入口

***

## 实施步骤

### 步骤一：创建用户管理模块 `src/utils/userManager.js`

* [ ] `createUser(username, password)` - 管理员创建用户

* [ ] `validateUser(username, password)` - 验证登录

* [ ] `getUserData(username)` - 获取用户数据

* [ ] `getAllUsers()` - 获取所有用户列表

* [ ] `deleteUser(username)` - 删除用户

* [ ] `updateUserPermission(username, permission)` - 更新用户权限

* [ ] `hasAssessmentAccess(username)` - 检查用户是否有学情评估权限

### 步骤二：创建认证上下文 `src/context/AuthContext.jsx`

* [ ] `currentUser` - 当前登录用户

* [ ] `isAdmin: boolean` - 是否管理员（用户名===ccb）

* [ ] `login(username, password)` - 登录

* [ ] `logout()` - 登出

* [ ] `hasAssessmentAccess()` - 当前用户是否有学情评估权限

### 步骤三：创建登录页面 `src/pages/LoginPage.jsx`

* [ ] 用户名输入框

* [ ] 密码输入框

* [ ] 登录按钮

* [ ] 错误提示（用户不存在/密码错误）

* [ ] **无注册入口**

### 步骤四：创建管理员面板 `src/components/admin/AdminPanel.jsx`

* [ ] 创建用户表单（用户名、初始密码）

* [ ] 用户列表（显示用户名、学情评估权限状态）

* [ ] 开通/关闭学情评估按钮

### 步骤五：创建管理员密码入口 `src/components/admin/AdminLoginModal.jsx`

* [ ] 管理员密码输入框（密码：admin2026）

### 步骤六：修改首页 `src/App.jsx`

* [ ] 检查登录状态，未登录跳转登录页

* [ ] 添加管理员入口（角落小图标）
