# "解题轨迹锁定"机制评估报告

## 问题分析

### 当前问题
用户反馈解析显示为 JSON 格式：
```
📖 解析：
{"core_idea":"新高考基础题特点...","key_steps":[...],"common_pitfalls":[...],"conclusion":"..."}
```

### 根本原因
1. **AI 输出格式错误**：AI 输出的是 JSON 对象，而不是格式化的文本
2. **解析显示逻辑问题**：`analysis` 字段被当作字符串直接显示，没有解析 JSON 结构
3. **因子与解析解耦**：AI 在生成解析时，没有明确知道它选了哪组参数

---

## 建议方案评估

### 方案 A：一步到位（推荐）✅

**核心思路**：让 AI 在一次回复中，先列出选的数，再出题，再写解析。

**优点**：
1. 解析紧跟着选题逻辑，不会出现幻觉
2. 因子选择和解析生成在同一上下文中
3. 减少一次 API 调用

**缺点**：
1. 需要修改 Prompt 结构
2. 输出格式需要重新设计

### 方案 B：分两步但传递因子

**核心思路**：在解析生成时，把选定的因子作为额外参数传入。

**优点**：
1. 兼容现有架构
2. 改动较小

**缺点**：
1. 需要两次 API 调用
2. 需要额外存储因子选择结果

---

## 实施建议

### 立即修复：解析显示问题

**问题**：`analysis` 字段是 JSON 字符串，但 UI 直接显示为文本。

**修复**：在 `LatexRenderer` 或显示逻辑中，检测 `analysis` 是否为 JSON，如果是则解析并格式化显示。

### 中期优化：采用方案 A

修改 Prompt 结构，让 AI 在一次回复中完成：
1. Step 1: 因子锁定（声明选了哪些参数）
2. Step 2: 题目生成
3. Step 3: 解析生成（严格绑定因子）

---

## 修复步骤

### 第一步：修复解析显示

在 `TaskCard` 组件中，检测 `analysis` 是否为 JSON 格式：

```javascript
const formatAnalysis = (analysis) => {
  if (!analysis) return ''
  
  // 尝试解析 JSON
  try {
    const parsed = typeof analysis === 'string' ? JSON.parse(analysis) : analysis
    
    // 格式化显示
    let formatted = ''
    if (parsed.core_idea) {
      formatted += `【核心思路】${parsed.core_idea}\n\n`
    }
    if (parsed.key_steps && Array.isArray(parsed.key_steps)) {
      formatted += `【关键步骤】\n${parsed.key_steps.map((s, i) => `${i + 1}. ${s}`).join('\n')}\n\n`
    }
    if (parsed.common_pitfalls && Array.isArray(parsed.common_pitfalls)) {
      formatted += `【易错警示】\n${parsed.common_pitfalls.map(p => `• ${p}`).join('\n')}\n\n`
    }
    if (parsed.conclusion) {
      formatted += `【结论】${parsed.conclusion}`
    }
    return formatted || analysis
  } catch (e) {
    // 不是 JSON，直接返回原文
    return analysis
  }
}
```

### 第二步：修改 Prompt 结构

采用方案 A，让 AI 在一次回复中完成因子锁定、题目生成、解析生成：

```
Task
请执行以下三个步骤，生成完整的题目及解析包：

Step 1: 因子锁定
- 针对第 (1) 问，从因子池中挑选符合 {levelBase} 复杂度的参数组合，记为 Set_A。
- 针对第 (2) 问，从因子池中挑选符合 {levelTarget} 复杂度的参数组合，记为 Set_B。
- 关键检查: 确认 Set_A 和 Set_B 的参数在数学逻辑上是成立的。

Step 2: 题目生成
- 基于 Set_A 和 Set_B，编写一道包含 (1)、(2) 两问的数学题。

Step 3: 解析生成 (严格绑定因子)
- 必须基于 Step 1 中选定的 Set_A 和 Set_B 进行推导！
- 禁止重新假设数值。
- 禁止使用通用模板套话。
```

### 第三步：更新输出格式

```
<output>
**题目**：...

**解析**：
【核心思路】一句话点破本题考察的本质
【关键步骤】
1. 代入 Set_A 的数值 x=..., y=...，计算得...
2. ...
【易错警示】结合本题数值的特定陷阱
【答案】...
</output>
```

## 结论

**建议方案合理，建议分两步实施：**

1. **立即修复**：解析显示问题（检测 JSON 并格式化）
2. **中期优化**：采用方案 A，一步到位生成题目和解析

**核心改进点**：
- 建立 "因子 -> 题目 -> 解析" 的强数据链
- 让 AI 在同一上下文中完成因子选择和解析生成
- 确保解析中的步骤严格绑定选定的参数
