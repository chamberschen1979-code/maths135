## 问题分析

根据您确认的规则，当前代码**规则逻辑是正确的**，但存在两个关键问题：

### 1. 题目抽取不随机
当前使用 `problemIndex % 数组长度` 的确定性方式选择题目，导致每次生成完全相同。

### 2. 缺少"重新生成"按钮
用户无法刷新已生成的题目。

## 修复方案

### 1. 修改 `src/utils/dataLoader.js`
将 `selectQuestionFromPool` 改为真正的随机选择：
```javascript
// 修改前
const selected = availableQuestions[problemIndex % availableQuestions.length]

// 修改后
const randomIndex = Math.floor(Math.random() * availableQuestions.length)
const selected = availableQuestions[randomIndex]
```

### 2. 修改 `src/utils/problemLogic.js`
- `selectBenchmark` 函数：将所有 `problemIndex % length` 改为随机选择
- `selectVariableKnobs` 函数：改为随机选择

### 3. 修改 `src/components/weekly/TaskGenerator.jsx`
添加"重新生成"按钮，当已有任务时显示。

### 4. 修改 `src/components/WeeklyMissionNew.jsx`
支持重新生成功能（清空现有任务后重新生成）

✅ 自选母题按选择范围抽取
