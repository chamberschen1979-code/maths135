# 数学无忧系统部署方案

## 问题回答

**是的，如果想让别人在网上使用这个系统，您需要：**
1. **域名**（可选但推荐）- 让用户通过易记的网址访问

## 部署方案选择

### 方案一：静态托管（推荐，免费或低成本）

由于这是一个纯前端 React 应用，最简单的方式是使用静态托管服务：

| 平台 | 费用 | 优点 | 缺点 |
|------|------|------|------|
| **Vercel** | 免费 | 部署简单、自动CI/CD、国内访问较快 | 需要绑定域名才能稳定访问 |
| **Netlify** | 免费 | 功能丰富、表单处理 | 国内访问可能较慢 |
| **GitHub Pages** | 免费 | 完全免费、与GitHub集成 | 只能部署公开仓库、功能有限 |
| **Cloudflare Pages** | 免费 | 全球CDN、速度快 | 配置稍复杂 |

### 方案二：国内云服务（稳定，需要备案）

| 平台 | 费用 | 说明 |
|------|------|------|
| **阿里云 OSS + CDN** | 约 ¥100-300/年 | 静态托管，需域名备案 |
| **腾讯云 COS + CDN** | 约 ¥100-300/年 | 静态托管，需域名备案 |
| **七牛云** | 有免费额度 | 适合小型应用 |

### 推荐方案：Vercel + 域名

**费用估算**：Vercel 免费 + 域名 ¥50-100/年 = **¥50-100/年**

## 开发调整后是否需要重新部署？

### 使用 Vercel/Netlify：**自动部署，无需手动操作**

```
本地修改代码 → git push → GitHub 收到代码 → 自动触发 Vercel 构建 → 自动部署上线
```

**整个过程约 1-2 分钟，完全自动！**

### 预览功能

Vercel 还提供**预览环境**：
- 每个 Pull Request 会生成一个预览链接
- 可以在合并前预览效果
- 正式环境不受影响

## 数据持久化方案

### 当前系统数据存储方式

- **题目数据**：存储在 JSON 文件中，打包时静态加载
- **学习进度**：存储在浏览器 localStorage 中
- **问题**：清除浏览器数据会丢失进度，换设备无法同步

### 数据持久化方案对比

| 方案 | 费用 | 复杂度 | 适合场景 |
|------|------|--------|----------|
| **Supabase** | 免费额度大 | 低 | 推荐，快速接入 |
| **Firebase** | 免费额度 | 低 | Google 生态 |
| **LeanCloud** | 有免费额度 | 低 | 国内服务，速度快 |
| **自建后端** | 需服务器 | 高 | 完全自主控制 |

### 推荐方案：Supabase

**Supabase** 是开源的 Firebase 替代品，提供：
- PostgreSQL 数据库
- 用户认证（登录/注册）
- 实时数据同步
- 行级安全策略

**免费额度**：
- 500MB 数据库
- 1GB 文件存储
- 50,000 月活用户

### 实施步骤（使用 Supabase）

1. **注册 Supabase**（supabase.com）
2. **创建项目**，获取 API 密钥
3. **创建数据表**：
   ```sql
   -- 用户学习进度表
   CREATE TABLE learning_progress (
     id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
     user_id UUID REFERENCES auth.users(id),
     motif_id TEXT,
     spec_id TEXT,
     var_id TEXT,
     level TEXT,
     consecutive_correct INTEGER DEFAULT 0,
     is_mastered BOOLEAN DEFAULT false,
     updated_at TIMESTAMP DEFAULT NOW()
   );
   ```
4. **安装 Supabase 客户端**：
   ```bash
   npm install @supabase/supabase-js
   ```
5. **修改代码**，将 localStorage 替换为 Supabase API

### 代码改动量估算

| 模块 | 改动量 |
|------|--------|
| 用户认证 | 新增登录/注册页面 |
| 数据存储层 | 重构 localStorage 相关代码 |
| 数据同步 | 新增实时同步逻辑 |
| **总计** | 约 500-1000 行代码 |

### 简化方案：仅同步关键数据

如果不想大改，可以只同步关键数据：
- 用户注册/登录
- 学习进度（掌握的题目、连续正确次数）
- 错题本

这样改动量更小，约 200-300 行代码。

## 实施步骤总结

### 第一步：部署静态网站
1. 将代码推送到 GitHub
2. 注册 Vercel 并导入项目
3. 配置构建命令：`npm run build`
4. 配置输出目录：`dist`
5. 点击部署
6. （可选）购买域名并绑定

### 第二步：（可选）添加数据持久化
1. 注册 Supabase
2. 创建数据表
3. 修改代码接入 API
4. 测试并部署

---

## 注意事项

1. **国内访问**：如果用户主要在国内，建议使用国内云服务或购买国内域名并备案
2. **数据持久化**：当前系统数据存储在浏览器 localStorage，清除浏览器数据会丢失进度
3. **HTTPS**：Vercel/Netlify 自动提供 HTTPS，确保数据传输安全
