# 双难度双问生成器优化方案评估报告

## 建议概述

将出题逻辑升级为"双难度双问生成器"（Dual-Level Dual-Question Generator）：
- **第(1)问**：levelBase（错题难度/用户选择难度）→ 解决"会不会"
- **第(2)问**：levelTarget（ELO能力难度）→ 解决"熟不熟、深不深"

---

## 核心设计分析

### 1. 双问结构的合理性 ✅

**优点**：
- 符合高考解答题标准格式（入口宽、出口深）
- 第(1)问保底，第(2)问拔高
- 学员即使做不出第(2)问，也能从第(1)问获得成就感

**数据支持**：
从 M01.json 可以看到，标杆题（master_benchmarks）确实存在多问结构：
```
"desc": "[24 新高考 I] 已知集合 A={x|x^2-3x+2=0}, B={1, 2, 3}，求 A∩B。"
```

**问题**：
- 现有 `master_benchmarks` 数据结构没有明确区分"第(1)问"和"第(2)问"
- 需要扩展数据结构，增加 `question_parts` 字段

### 2. 双难度分层的合理性 ✅

**场景分析**：

| 场景 | levelBase | levelTarget | 效果 |
|------|-----------|-------------|------|
| 错题重练（L2错题，ELO=L3） | L2 | L3 | 修基础+拔高 |
| 手动勾选（选L2，ELO=L4） | L2 | L4 | 保底+挑战 |
| 手动勾选（选L2，ELO=L2） | L2 | L2 | 稳扎稳打 |
| ELO推荐（薄弱点，ELO=L2） | L2 | L2/L3 | 直面弱点 |

**优点**：
- 解决了"错题难度被ELO覆盖"的问题
- 兼顾"修复错题"和"能力拔高"两个目标

**问题**：
- 当 levelBase == levelTarget 时，如何区分两问的难度？
- 建议方案说"侧重点不同"，但AI可能难以理解

### 3. 三策略定义的合理性 ⚠️

**建议方案定义**：
```
Q1_CALC_FOCUS: 侧重计算稳健性
Q2_LOGIC_FOCUS: 侧重思维灵活性
Q3_STRUCT_FOCUS: 侧重结构严谨性
```

**问题**：
1. **抽象概念难以落地**：AI可能无法准确理解"计算稳健性"vs"思维灵活性"的区别
2. **因子选择逻辑缺失**：`selectComplex(pool)`、`selectSimple(pool)` 函数没有具体实现
3. **三道题可能重复**：如果AI理解偏差，三道题可能变成"换数字"

**改进建议**：
将抽象策略转化为具体的Prompt约束：
```
Q1: 第(1)问使用整数参数，第(2)问使用分数参数
Q2: 第(1)问正向提问，第(2)问逆向提问
Q3: 第(1)问显性条件，第(2)问隐含条件
```

## 实施建议

### 第一阶段：Prompt层增强（低成本）

在现有Prompt中增加双问约束：
```
【双问结构要求】
- 第(1)问：难度锁定【L2】，目的是修复基础
- 第(2)问：难度提升至【L3】，目的是能力拔高
- 两问之间可以有逻辑依赖（如第(2)问用到第(1)问的结论）
```

### 第二阶段：数据结构扩展（中成本）

1. 为 `master_benchmarks` 增加 `question_parts` 字段
2. 定义 `benchmark_structure` 结构
3. 修改 `generateAIProblem` 函数，支持双问生成

### 第三阶段：完整重构（高成本）

1. 实现 `resolveSampleContext` 上下文解析器
2. 实现 `generateDualLevelTriad` 双难度生成器
3. 实现因子复杂度分析函数

---

## 结论

**建议方案整体合理，但需要分阶段实施：**

| 评估项 | 评分 | 说明 |
|--------|------|------|
| 双问结构 | ⭐⭐⭐⭐⭐ | 符合高考格式，教育价值高 |
| 双难度分层 | ⭐⭐⭐⭐ | 解决了错题难度被覆盖的问题 |
| 三策略定义 | ⭐⭐⭐ | 抽象概念需转化为具体约束 |
| 数据结构兼容 | ⭐⭐⭐ | 需要扩展 benchmark_structure |
| 实施难度 | ⭐⭐⭐ | 中等，可分阶段实施 |

**核心改进点**：
1. 先在Prompt层面实现双问约束
2. 验证效果后再考虑数据结构扩展
3. 将抽象策略转化为具体的因子选择规则
