# 录入新题功能优化计划

## 📊 用户需求分析

### 1. 增加母题/专项/变例的选择
- 当前：母题/专项/变例是固定传入的
- 需求：用户可以下拉选择母题、专项、变例

### 2. 保留难度选择
- 当前：已有 L2/L3/L4 难度选择
- 需求：保留此功能

### 3. 删除"变例信息"区域
- 当前：显示"母题：xxx"、"变例：xxx"、"ID 前缀：xxx"
- 需求：删除此区域

### 4. AI 填充环节
- 问题：用户录入的字段和题库不一致
- 题库字段：`id`, `problem`, `answer`, `key_points`, `level`, `tags`, `meta`, `analysis`, `specId`, `specName`, `varId`, `varName`
- 用户录入字段：`problem`, `answer`, `keyPoints`, `level`
- 需求：使用 AI 补充完整字段

### 5. 撤回功能
- 需求：入库的题目可以撤回（删除）

### 步骤 3：添加 AI 填充环节

**新增 AI 填充按钮**：
- 点击后调用 AI 接口
- AI 分析题目内容，补充以下字段：
  - `tags`：题目标签
  - `meta.core_logic`：核心逻辑
  - `meta.trap_tags`：易错点
  - `meta.weapons`：关联杀手锏
  - `analysis`：详细解析

**AI Prompt 示例**：
```
分析以下数学题目，补充元数据：
题目：{problem}
答案：{answer}
母题：{motifName}
变例：{varName}

请返回：
1. tags: 题目标签数组
2. core_logic: 核心逻辑数组
3. trap_tags: 易错点数组
4. weapons: 关联杀手锏ID数组
5. analysis: 详细解析
```

### 步骤 4：添加撤回功能

**方案**：
1. 在 `UserProgressContext` 中添加 `removeFromUserPool` 方法
2. 在录入成功后显示"撤回"按钮（限时 30 秒）
3. 或者在"我的题库"页面添加删除功能

**代码示例**：
```jsx
// UserProgressContext.jsx
const removeFromUserPool = (questionId) => {
  setUserPool(prev => prev.filter(q => q.id !== questionId));
};

// ManualEntryModal.jsx
const [lastAddedId, setLastAddedId] = useState(null);
const [showUndo, setShowUndo] = useState(false);

const handleUndo = () => {
  if (lastAddedId) {
    removeFromUserPool(lastAddedId);
    setShowUndo(false);
  }
};
```
