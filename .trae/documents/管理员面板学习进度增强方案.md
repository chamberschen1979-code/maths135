# 管理员面板学习进度增强方案

## 现状分析

### 现有数据架构

| 数据类型 | 存储位置 | 说明 |
|---|---|---|
| 变例基准状态（ELO、gear_level） | `user_${username}_tactical_data` | 来自 `tacticalMaps.json` 的初始数据 |
| 用户掌握状态 | 同上 | `encounters[].specialties[].variations[].master_benchmarks[].is_mastered` |
| 杀手锏认证 | 同上 | `user_profile.certifiedWeapons[]` |
| 做题历史 | `user_${username}_question_history` | 每题 `isMastered`、`consecutiveCorrect`、`lastGrade` |
| 每周任务 | `user_${username}_weekly_tasks` | 任务生成记录 |

**关键文件**：
- `src/hooks/useTrainingCenterData.js` — 统一起订数据源，复用 Mxx.json 的 benchmark 定义，结合用户存储的 `is_mastered` 状态
- `src/components/HoloMap.jsx` — 学生端"变例通关"逻辑（**变例下所有 L2/L3/L4 全部点亮才算通关**）
- `src/components/training/MiniStatsBar.jsx` — 显示 `masteredCount/totalCount`（benchmark 级别）

**没有 ELO 历史**：当前只存当前 ELO，无变化曲线
**没有活动日志**：每次完成战斗/认证无自动埋点

## 实施步骤

### Step 1：复用 useTrainingCenterData 统一口径

`useTrainingCenterData.js` 已有完整计算逻辑：
- `masteredCount` = 已点亮绿灯的 benchmark 总数
- `totalCount` = 所有 benchmark 总数（≈174）
- `nearCompleteCount` = 即将通关（streak ≥ 1 且 < 3）
- `meltdownCount` = 红灯数（is_locked 或 l2_status === 'RED'）

AdminPanel 直接调用此 hook 即可获得一致的口径。

### Step 2：新建 activity_log 数据层

在 `src/utils/userManager.js` 中新增：

```js
// activity_log 每条记录结构
{
  id: string,       // `${Date.now()}_${Math.random().toString(36).slice(2)}`
  type: 'battle' | 'certification' | 'assessment' | 'error_resolved',
  timestamp: ISO string,
  motifId?: string,  // battle 时
  varId?: string,    // battle 时
  grade?: string,    // 评级 S/A/B/C
  eloDelta?: number,
  weaponId?: string, // certification 时
  description: string
}
```

新增函数：
- `addActivityLog(username, entry)` — 追加一条
- `getActivityLog(username, days)` — 获取最近 N 天记录
- `getActivityStats(username, days)` — 统计：做题数、准确率、ELO 变化

### Step 3：埋点写入

| 场景 | 位置 | 埋点类型 | 记录字段 |
|---|---|---|---|
| 战斗结算 | `handleBattleComplete` 末尾 | `battle` | motifId, varId, grade, eloDelta |
| 杀手锏认证 | `handleWeaponCertified` 末尾 | `certification` | weaponId |
| 错题解决 | `resolveError` 调用处 | `error_resolved` | motifId, varId |

### Step 4：AdminPanel 学习进度页重构

**交互**：默认展开第一个用户

**复用 useTrainingCenterData**（新开 hook 实例传入目标用户的 tacticalData）

**四大指标卡片**：

1. **绿灯率**
   - 直接用 `masteredCount/totalCount`
   - 进度条 + 百分比
   - 同时显示 `nearCompleteCount`（即将通关）

2. **ELO 战力**
   - 当前总 ELO、平均 ELO
   - 若有 ELO 历史：显示 ΔELO（7日）

3. **本周训练强度**
   - 从 activity_log 统计最近 7 天的 battle 记录数
   - 从 question_history 统计准确率
   - 周均 ELO 变化

4. **杀手锏认证**
   - 已认证数 / 总数
   - 最近认证的杀手锏列表（取前5个）

**Pain Point**：`nearCompleteCount` 和 `meltdownCount` 可直接用于识别即将通关但卡住的、以及持续红灯的变例。
