# AI 解析生成优化方案：两阶段输出架构

## 问题分析

### 当前问题
用户反馈：展示区（analysis）仍然出现内心思考过程，AI 似乎不理解"内心推理"和"展示解析"的区别。

### 根本原因
1. **AI 理解困难**：单个 Prompt 中同时要求两种不同风格的输出，AI 难以区分
2. **输出混淆**：`thought_process` 和 `analysis` 的边界模糊，AI 倾向于把推理过程写进 `analysis`
3. **格式不一致**：AI 输出的 LaTeX 公式经常忘记用 `$...$` 包裹

## 当前做法 vs 方案A 对比

### 当前做法（JSON 字段）
```
输出格式：
{
  "thought_process": "AI内心推理（不展示）",
  "question": "题目",
  "analysis": "解析（展示）",
  "answer": "答案"
}
```

**问题**：
- 4个字段是平级的，AI 容易混淆 `thought_process` 和 `analysis`
- JSON 字段名只是标签，AI 不理解语义差异
- `analysis` 字段名没有明确表示"这是给学生看的"

### 方案A（XML 标签）
```
输出格式：
<reasoning>
这里是完整推理过程（不展示）
</reasoning>

<output>
这里是展示给学生的内容
**题目**：...
**解析**：...
**答案**：...
</output>
```

**改进**：
- 使用 XML 标签，格式更明确，AI 更容易理解边界
- `<reasoning>` = 草稿区（不展示），`<output>` = 展示区（展示）
- **没有取消草稿区**，只是改用 XML 标签替代 JSON 字段

### 关键区别总结

| 对比项 | 当前做法 | 方案A |
|--------|----------|-------|
| 格式 | JSON 字段 | XML 标签 |
| 草稿区 | `thought_process` 字段 | `<reasoning>` 标签 |
| 展示区 | `analysis` 字段 | `<output>` 标签 |
| AI 理解度 | 字段名语义模糊 | 标签边界明确 |
| 格式约束 | JSON Schema | 明确的输出模板 |

## 行业解决方案研究

### 思维链（Chain of Thought）技术
根据搜索结果，思维链（CoT）技术的核心是：
> "引导模型在输出最终答案之前，先生成一系列中间推理步骤"

这正是我们需要的！但关键区别是：
- **CoT 的推理过程**：用于提升答案准确性，**不展示给用户**
- **最终输出**：精炼后的结果，**展示给用户**

### 业界最佳实践
1. **两阶段生成**：先生成完整推理，再精炼输出
2. **格式约束**：使用严格的 JSON Schema 或 XML 标签
3. **角色分离**：让 AI 扮演不同角色（出题者 vs 解析者）

## 推荐方案：两阶段输出架构

### 方案概述
```
阶段1: AI 完整推理（不展示）
   ↓
阶段2: 精炼输出（展示给学生）
```

### 具体实现

#### 方案A：单次调用 + 格式分离（推荐）

**Prompt 结构：**
```
你是一个高中数学出题专家。请按以下格式输出：

<reasoning>
【这里是你的完整推理过程】
- 分析题目考点
- 设计参数
- 验证答案
- 这个内容不会展示给学生，可以自由发挥
</reasoning>

<output>
【这里是展示给学生的内容】
严格按照以下格式，不要包含任何内心思考：

**题目**：$...$（所有公式用$包裹）

**解析**：
【核心思路】一句话点破关键
【关键步骤】
1. 第一步...
2. 第二步...
3. 第三步...

**答案**：$...$
</output>
```

**代码处理：**
```javascript
// 提取 <reasoning> 内容（不展示）
const reasoningMatch = aiContent.match(/<reasoning>([\s\S]*?)<\/reasoning>/i)
const reasoning = reasoningMatch ? reasoningMatch[1] : ''

// 提取 <output> 内容（展示）
const outputMatch = aiContent.match(/<output>([\s\S]*?)<\/output>/i)
const output = outputMatch ? outputMatch[1] : ''
```

#### 方案B：两次调用（更精确但更慢）

**第一次调用：生成完整推理**
```
请生成一道高中数学题，并写出你的完整推理过程。
输出格式：JSON { "reasoning": "...", "question": "...", "answer": "..." }
```

**第二次调用：精炼解析**
```
请将以下推理过程精炼为学生版解析，要求：
- 只保留关键步骤
- 使用标准格式：【核心思路】...【关键步骤】1. 2. 3.
- 所有公式用 $...$ 包裹

推理过程：
{第一次调用的 reasoning}

请输出精炼后的解析：
```

### 推荐采用方案A

**优点：**
1. 单次 API 调用，速度快
2. 使用 XML 标签明确分离两种内容
3. AI 更容易理解格式要求

**关键改进点：**
1. 使用 `<reasoning>` 和 `<output>` 标签替代 JSON 字段
2. 在 `<output>` 中强调"不要包含任何内心思考"
3. 提供明确的格式模板

## 实施步骤

1. **修改 Prompt 结构**
   - 使用 XML 标签替代 JSON 字段
   - 明确区分 `<reasoning>` 和 `<output>`

2. **修改代码解析逻辑**
   - 使用正则提取 `<reasoning>` 和 `<output>` 内容
   - `<reasoning>` 存储但不展示
   - `<output>` 展示给学生

3. **增强格式约束**
   - 在 `<output>` 部分强调 LaTeX 格式要求
   - 提供标准输出模板

4. **测试验证**
   - 重新生成题目
   - 检查解析是否不再包含内心思考
   - 检查 LaTeX 格式是否正确

## 预期效果

| 阶段 | 内容 | 是否展示 |
|------|------|----------|
| `<reasoning>` | AI 完整推理过程 | ❌ 不展示 |
| `<output>` | 精炼后的学生版解析 | ✅ 展示 |

**示例输出：**
```
<reasoning>
这道题考查复数方程求解，我选择参数 a=2 是因为...
首先计算判别式 Δ=72>0...
验证答案正确性...
</reasoning>

<output>
**题目**：已知集合 $A=\{z \in \mathbb{C} \mid z^2-4z+5=0\}$...

**解析**：
【核心思路】利用求根公式解复数方程，注意判别式为负时根为共轭复数。
【关键步骤】
1. 计算判别式 $\Delta = 16-20 = -4 < 0$
2. 得两共轭复根 $z = 2 \pm i$
3. 验证 $|z| = \sqrt{5}$，确认 $A \cap B = A$

**答案**：$A \cap B = \{2+i, 2-i\}$
</output>
```
