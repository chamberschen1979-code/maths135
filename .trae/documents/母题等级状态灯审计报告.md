# 母题等级状态灯审计报告

## 一、问题分析

### 用户问题
"列表每个母题后面只有L2/L3两个级别加状态灯，要加上L4+状态灯。此外，这个是写死的，还是动态抓取的？"

### 答案：**动态抓取，不是写死**

---

## 二、当前实现逻辑

### 2.1 数据源：`tacticalMaps.json`

每个母题的 `sub_targets` 数组决定了显示哪些等级的状态灯：

```json
{
  "target_id": "M01",
  "sub_targets": [
    { "sub_id": "M01_L2_1", "level_req": "L2" },
    { "sub_id": "M01_L3_1", "level_req": "L3" }
    // ❌ 没有 L4
  ]
}
```

### 2.2 代码逻辑：`TacticalDashboard.jsx` 第437-443行

```javascript
const l2Subs = encounter.sub_targets?.filter(s => s.level_req === 'L2') || []
const l3Subs = encounter.sub_targets?.filter(s => s.level_req === 'L3') || []
const l4Subs = encounter.sub_targets?.filter(s => s.level_req === 'L4') || []

const l2Status = getLevelStatus(l2Subs)
const l3Status = getLevelStatus(l3Subs)
const l4Status = getLevelStatus(l4Subs)
```

### 2.3 UI 渲染：第470-487行

```jsx
{l2Subs.length > 0 && (
  <div>L2 熟练 <span className={getStatusDotColor(l2Status)}></span></div>
)}
{l3Subs.length > 0 && (
  <div>L3 迁移 <span className={getStatusDotColor(l3Status)}></span></div>
)}
{l4Subs.length > 0 && (
  <div>L4 融会 <span className={getStatusDotColor(l4Status)}></span></div>
)}
```

**结论**：只有当 `l4Subs.length > 0` 时才会显示 L4 状态灯。

## 四、问题根源

**不是代码写死，而是数据不完整。**

`tacticalMaps.json` 中只有 5 个母题有 L4 级别的 `sub_targets`，其他 12 个母题缺少 L4 数据。

---

## 五、修复方案

### 方案：补全 `tacticalMaps.json` 中缺失的 L4 子目标

需要为以下 12 个母题添加 L4 级别的 `sub_targets`：

