# 母题等级状态灯修复方案

## 一、用户观点（正确）

1. **每个母题都有 L2/L3/L4**：等级结构是固定的
2. **sub_targets 可以写死**：因为等级结构是固定的
3. **状态需要动态读取**：`is_mastered` 字段需要从用户进度数据中读取

## 二、当前问题

### 2.1 tacticalMaps.json 数据不完整

当前 `tacticalMaps.json` 中，基础母题（M01-M10）的 `sub_targets` 只定义到 L3，缺少 L4。

### 2.2 代码逻辑

```javascript
// 当前逻辑：只有当 sub_targets 中有对应等级时才显示状态灯
const l4Subs = encounter.sub_targets?.filter(s => s.level_req === 'L4') || []
{l4Subs.length > 0 && (
  <div>L4 融会 <状态灯 /></div>
)}
```

**问题**：如果 `sub_targets` 没有 L4 数据，状态灯就不会显示。

## 三、修复方案

### 方案 A：补全 tacticalMaps.json 数据

为所有 17 个母题添加完整的 L2/L3/L4 `sub_targets`：

```json
{
  "target_id": "M01",
  "sub_targets": [
    { "sub_id": "M01_L2_1", "sub_name": "集合运算", "level_req": "L2", "is_mastered": false },
    { "sub_id": "M01_L3_1", "sub_name": "集合综合", "level_req": "L3", "is_mastered": false },
    { "sub_id": "M01_L4_1", "sub_name": "集合压轴", "level_req": "L4", "is_mastered": false }
  ]
}
```

### 方案 B：代码层面写死等级结构

修改 `TacticalDashboard.jsx`，不依赖 `sub_targets` 的 `level_req`，而是直接显示 L2/L3/L4：

```javascript
// 写死等级结构
const LEVELS = ['L2', 'L3', 'L4']

// 动态读取状态
const getLevelStatus = (encounter, level) => {
  const subs = encounter.sub_targets?.filter(s => s.level_req === level) || []
  if (subs.length === 0) return 'locked'
  if (subs.every(s => s.is_mastered === true)) return 'mastered'
  if (subs.some(s => s.is_mastered === false)) return 'practicing'
  return 'locked'
}

// UI 渲染
{LEVELS.map(level => (
  <div key={level}>
    {level} {getLevelStatusLabel(level)}
    <span className={getStatusDotColor(getLevelStatus(encounter, level))}></span>
  </div>
))}
```

## 四、推荐方案

**采用方案 A + 方案 B 结合**：

1. **补全 tacticalMaps.json**：确保所有母题都有 L2/L3/L4 的 `sub_targets`
2. **代码层面写死等级结构**：不依赖 `sub_targets` 是否有对应等级，直接显示 L2/L3/L4
3. **状态动态读取**：`is_mastered` 状态从用户进度数据中读取
