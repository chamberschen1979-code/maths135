## 二、设计思考

### 列表模块的存在意义

**存在的意义**：
1. **快速概览**：一览所有母题的掌握状态
2. **筛选定位**：快速找到薄弱点
3. **训练入口**：直接跳转到训练

**是否显示专项和变例**：

| 方案 | 优点 | 缺点 |
|------|------|------|
| 只显示母题 | 简洁、一目了然 | 无法看到细节进度 |
| 显示母题+专项 | 可以看到专项进度 | 信息量增加 |
| 显示母题+专项+变例 | 最详细 | 信息过载，难以浏览 |

**建议方案**：**分层展示**
- 默认显示母题（折叠状态）
- 点击展开显示专项
- 再点击展开显示变例

### 错题库位置

**当前问题**：
- 错题库入口隐藏在右上角按钮，不够显眼
- 错题诊断页面主要功能是AI对话，错题库是次要功能

**建议方案**：**Tab切换模式**

```
┌─────────────────────────────────────────────────────┐
│  [AI诊断]  [错题库]                    ← Tab切换     │
├─────────────────────────────────────────────────────┤
│                                                     │
│  AI诊断Tab:                                          │
│  - AI对话区                                          │
│  - 拍照诊断                                          │
│                                                     │
│  错题库Tab:                                          │
│  - 筛选器                                            │
│  - 错题列表                                          │
│  - 导入/导出功能                                     │
│                                                     │
└─────────────────────────────────────────────────────┘
```

### 经验值卡片

**当前状态**：
- 定义在 App.jsx:40 的 EXP_MAP
- 在战斗结算弹窗中显示

**用户意见**：删除

**建议**：删除经验值相关逻辑，简化评分系统（已在新Elo算法中简化）

## 三、重构方案

### 方案A：列表模块增强

**修改文件**：`TacticalDashboard.jsx`

**新增功能**：
1. 分层展示：母题 → 专项 → 变例
2. 每层显示对应的进度状态
3. 支持折叠/展开

**示例结构**：
```
▼ M16 排列组合（Elo: 1850）
  ├─ 🟢 染色问题专项
  │   ├─ 🟢 变例1.1: 直接染色
  │   └─ 🟡 变例1.2: 环形染色
  ├─ 🟡 路径问题专项
  │   ├─ 🔴 变例2.1: 网格路径
  │   └─ 🟢 变例2.2: 最短路径
  └─ ...
```

### 方案B：错题库Tab化

**修改文件**：`App.jsx` 或新建 `DiagnosisView.jsx`

**新增功能**：
1. Tab切换：AI诊断 / 错题库
2. 错题库作为独立Tab，功能完整展示
3. 删除经验值卡片相关代码

### 方案C：删除经验值系统

**修改文件**：`App.jsx`

**删除内容**：
- EXP_MAP 定义
- 经验值计算逻辑
- 结算弹窗中的经验值显示

## 六、风险评估

| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| 列表信息过载 | 用户难以浏览 | 默认折叠，按需展开 |
| Tab切换影响AI诊断使用频率 | 用户可能忽略AI功能 | 保持AI诊断为默认Tab |
| 删除经验值影响用户激励 | 用户失去成就感 | Elo分数已提供足够激励 |
