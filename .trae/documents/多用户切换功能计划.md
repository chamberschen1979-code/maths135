# 多用户切换功能计划

## 📋 需求分析

当前系统所有数据存储在 localStorage 中，无用户隔离。需要实现：
- 创建新用户
- 切换用户
- 切换后保留原用户数据

| Storage Key | 用途 | 文件位置 |
|------------|------|---------|
| `weapon_progress` | 武器学习/认证进度 | weaponProgress.js |
| `tactical_data` | 战术数据（ELO、主题状态） | App.jsx |
| `tactical_data_version` | 数据版本号 | App.jsx |
| `error_notebook` | 错题本 | App.jsx |
| `weekly_plan` | 周计划 | App.jsx |
| `weekly_tasks` | 周任务 | App.jsx |

## 🔧 技术细节

### userManager.js 核心代码结构

```javascript
const USERS_KEY = 'users'
const CURRENT_USER_KEY = 'current_user_id'

export const getCurrentUser = () => {
  const userId = localStorage.getItem(CURRENT_USER_KEY)
  if (!userId) return null
  const users = getAllUsers()
  return users.find(u => u.id === userId)
}

export const createUser = (name) => {
  const users = getAllUsers()
  const newUser = {
    id: `user_${Date.now()}`,
    name,
    createdAt: new Date().toISOString(),
    lastActiveAt: new Date().toISOString()
  }
  users.push(newUser)
  localStorage.setItem(USERS_KEY, JSON.stringify(users))
  return newUser
}

export const switchUser = (userId) => {
  const users = getAllUsers()
  const user = users.find(u => u.id === userId)
  if (!user) return false
  
  // 更新最后活跃时间
  user.lastActiveAt = new Date().toISOString()
  localStorage.setItem(USERS_KEY, JSON.stringify(users))
  localStorage.setItem(CURRENT_USER_KEY, userId)
  
  return true
}

export const getUserStorageKey = (key) => {
  const user = getCurrentUser()
  if (!user) return key // 兼容旧数据
  return `${user.id}_${key}`
}
```

### 数据迁移策略

首次启动时：
1. 检测是否存在 `weapon_progress` 等旧数据
2. 创建默认用户 "默认用户"
3. 将旧数据迁移到 `user_xxx_weapon_progress` 等
4. 删除旧 key（可选，或保留备份）

## ✅ 验收标准

1. 可以创建新用户
2. 可以切换用户
3. 切换用户后，各用户数据独立
4. 切换回原用户，数据完整保留
5. 首次启动自动迁移旧数据
