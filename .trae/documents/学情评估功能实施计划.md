### 3.3 核心逻辑实现

#### 3.3.1 母题结果计算（木桶效应）

```javascript
// 计算母题结果（核心逻辑）
const calculateMotifResults = (results, tasks) => {
  // 按母题分组
  const motifMap = new Map();
  
  results.forEach((result, index) => {
    const task = tasks[index];
    const { motifId, variationId, variationName } = task;
    
    if (!motifMap.has(motifId)) {
      motifMap.set(motifId, {
        motifId,
        variations: [],
        totalVariations: 0
      });
    }
    
    const motifData = motifMap.get(motifId);
    motifData.variations.push({
      variationId,
      variationName,
      correct: result.correct,
      status: result.correct ? 'green' : 'red'
    });
    motifData.totalVariations++;
  });
  
  // 计算每个母题的整体结果
  return Array.from(motifMap.values()).map(motif => {
    const correctCount = motif.variations.filter(v => v.correct).length;
    const allCorrect = correctCount === motif.totalVariations;
    const allIncorrect = correctCount === 0;
    const partialCorrect = !allCorrect && !allIncorrect;
    
    let elo, level, overallStatus;
    
    if (allCorrect) {
      elo = 1801;
      level = 'L3准入';
      overallStatus = 'green';
    } else if (partialCorrect) {
      elo = 1400;
      level = 'L2进阶';
      overallStatus = 'yellow';
    } else {
      elo = 1001;
      level = 'L2基础';
      overallStatus = 'red';
    }
    
    return {
      ...motif,
      correctCount,
      elo,
      level,
      overallStatus
    };
  });
};
```

#### 3.3.2 战术数据更新

```javascript
// 更新tacticalData
const updateTacticalData = (motifResults, tacticalData, setTacticalData) => {
  const updatedData = { ...tacticalData };
  
  motifResults.forEach(result => {
    const { motifId, elo, variations } = result;
    
    if (!updatedData[motifId]) {

        elo: 1000,
        variations: {}
      };
    }
    
    // 更新母题ELO（木桶效应）
    updatedData[motifId].elo = elo;
    
    // 更新变例状态
    variations.forEach(variation => {

        updatedData[motifId].variations[variation.variationId] = {
          vid: variation.variationId,
          status: 'locked'
        };
      }
      updatedData[motifId].variations[variation.variationId].status = 
        variation.correct ? 'mastered' : 'learning';
    });
  });
  
  setTacticalData(updatedData);
};
```

## 八、红绿灯判定规则（含部分做对情况）

### 8.1 核心原则：木桶效应 + 精准打击

* **ELO计算**：母题整体等级由最弱变例决定（防止虚高）

* **红绿灯状态**：如实反映每个变例的掌握情况（方便精准推荐）

### 8.2 具体判定逻辑

| 测试情况 | 判定结果 | 初始ELO | 红绿灯状态   | 后续路径   |
| ---- | ---- | ----- | ------- | ------ |
| 全对   | L3准入 | 1801  | 全绿      | 解锁L3进阶 |
| 部分对  | L2进阶 | 1400  | 部分绿、部分红 | 定向补漏   |
| 全错   | L2基础 | 1001  | 全红      | 重修L2基础 |

### 8.3 变例级别判定

* 答对：L2、L3灯亮绿灯，L4灯灰

* 答错：L2、L3灯亮红灯，L4灯灰

### 8.4 母题级别判定

* **全对**：L2绿、L3绿、L4灰，ELO=1801

* **部分对**：L2绿、L3红、L4灰，ELO=1400

* **全错**：L2红、L3灰、L4灰，ELO=1001

### 8.5 为什么给1400分？

* **不是1001**：学生做对了一半，不是零基础，不需要从最浅显的概念学起

* **不是1801**：L3要求全面掌握，有变例做错说明知识网有破洞

* **1400的意义**："修补期"分数，承认基础，指出不足

## 九、结果页展示与任务推荐

### 9.1 结果页展示（变例级颗粒度）

```
M03 函数概念与性质：综合评分 1400 (L2 进阶)
✅ V1 结构基石：已掌握 (跳过)
❌ V2 对称周期：薄弱 (建议练习)
```

### 9.2 任务推荐逻辑（定向爆破）

1. **检测**：M03总体是L2状态，但V1是绿灯
2. **过滤**：在L2/L3题库中，只抽取V2变例的题目
3. **避免**：不推V1的简单题，避免浪费时间

### 9.3 升级机制（动态解锁）

1. 学生完成V2的专项练习（做对3道L2/L3题）
2. 系统检测到V2变绿
3. **触发**：M03母题整体状态自动升级为L3 (1801+)，解锁L4权限
