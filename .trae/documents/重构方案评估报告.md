## 二、promptBuilder.js 代码评估

### ✅ 优点

1. **Few-Shot 示例完美**
   ```javascript
   const PERFECT_OUTPUT_EXAMPLE = {
     reasoning: { q1_params: {...}, q2_params: {...}, verification: "..." },
     question: { content: "..." },
     analysis: { core_idea: "...", steps: [...] },
     answer: { l1: "...", l2: "..." }
   }
   ```
   - 结构清晰，字段完整
   - LaTeX 格式正确（使用 `$...$`）
   - 包含验证逻辑

2. **System Prompt 约束强**
   - OUTPUT FORMAT: 严格 JSON
   - LaTeX Hygiene: 详细规则
   - Reasoning Field Ban: 禁止 LaTeX
   - Logic Flow: 流程清晰

3. **User Prompt 动态注入**
   - 接收 context 参数
   - 包含母题、难度、参数池等信息

### ⚠️ 需要修复的问题

#### 问题 1：模板字符串转义错误

**原代码**：
```javascript
const knobsDescription = variableKnobs.map(k => 
  `- ${k.name} (权重: ${k.weight}): ${JSON.stringify(k.values)}`
).join('\n');
```

**问题**：在模板字符串中使用 `${}`，但外层也是模板字符串，会导致解析错误。

**修复**：
```javascript
const knobsDescription = variableKnobs.map(k => 
  `- ${k.name} (权重: ${k.weight}): ${JSON.stringify(k.values)}`
).join('\n');
```

#### 问题 2：JSON.stringify 在模板字符串中的处理

**原代码**：
```javascript
# Output Schema Example
You MUST follow this structure exactly:
${JSON.stringify(PERFECT_OUTPUT_EXAMPLE, null, 2)}
```

**问题**：JSON 中的反斜杠会被转义，导致 LaTeX 格式错误。

**修复**：需要确保 JSON 中的 LaTeX 格式正确。

#### 问题 3：缺少默认值处理

**原代码**：
```javascript
const { motifName, specName, varName, difficultyConfig, variableKnobs, benchmarkQuestion } = context;
```

**问题**：如果 context 中缺少某些字段，会导致 undefined。

**修复**：
```javascript
const {
  motifName = '未知母题',
  specName = '通用专项',
  varName = '常规变式',
  difficultyConfig = { level: 'L2', tier: '基础筑基', complexity: 1, steps: 2, traps: 0, paramConstraint: 'integer' },
  variableKnobs = [],
  benchmarkQuestion = null
} = context;
```

## 三、建议的修复方案

### 修复后的 promptBuilder.js

```javascript
// src/utils/promptBuilder.js

const PERFECT_OUTPUT_EXAMPLE = {
  reasoning: {
    q1_params: {
      values: "a=2, b=1",
      reason: "L2难度，选择小整数以确保计算简单，判别式为完全平方数"
    },
    q2_params: {
      values: "a=3, b=2",
      reason: "L3难度，引入稍大整数，需分类讨论根的符号"
    },
    verification: "Q1: x^2-2x+1=0 -> (x-1)^2=0, x=1 (OK). Q2: x^2-3x+2=0 -> x=1,2 (OK)."
  },
  question: {
    content: "（1）已知集合 $A = \\{x \\mid x^2 - ax + b = 0\\}$，若 $A = \\{1\\}$，求 $a, b$ 的值。\n（2）设方程 $x^2 - mx + n = 0$ 的两根为 $x_1, x_2$，当 $m=3, n=2$ 时，判断 $x_1, x_2$ 的符号。"
  },
  analysis: {
    core_idea: "利用韦达定理及集合元素的互异性/确定性求解参数。",
    steps: [
      "对于(1)：由 $A=\\{1\\}$ 知方程有唯一实根 $x=1$。",
      "根据韦达定理：$x_1+x_2=a \\Rightarrow 1+1=a \\Rightarrow a=2$。",
      "$x_1 \\cdot x_2=b \\Rightarrow 1 \\cdot 1=b \\Rightarrow b=1$。",
      "对于(2)：方程为 $x^2-3x+2=0$，解得 $x_1=1, x_2=2$。",
      "两者均为正数。"
    ]
  },
  answer: {
    l1: "$a=2, b=1$",
    l2: "$x_1 > 0, x_2 > 0$"
  }
};

export const buildSystemPrompt = () => {
  const exampleJson = JSON.stringify(PERFECT_OUTPUT_EXAMPLE, null, 2);
  
  return `# Role
你是一位资深高中数学命题专家，擅长根据给定的知识点、难度等级和参数约束，逆向构造高质量的数学题目。

# Critical Constraints (ABSOLUTE RULES)
1. **OUTPUT FORMAT**:
   - Output **STRICTLY** a single valid JSON object.
   - **NO** markdown code blocks (no \`\`\`json).
   - **NO** introductory or concluding text.
   - Start immediately with '{' and end with '}'.

2. **LaTeX Hygiene**:
   - Use SINGLE '$' for inline math: e.g., '$x^2$'.
   - **NEVER** double wrap: WRONG '$ $x^2$ $', RIGHT '$x^2$'.
   - **NEVER** put LaTeX commands outside '$': WRONG '\\{x \\mid x>0\\}', RIGHT '$\\{x \\mid x>0\\}$'.
   - In JSON strings, escape backslashes: use '\\\\frac', '\\\\sqrt', '\\\\mid'.

3. **Reasoning Field Ban**:
   - The 'reasoning' field must be **PLAIN TEXT ONLY**.
   - **NO** '$', **NO** '\\', **NO** LaTeX in 'reasoning'.
   - Example: "delta = b^2 - 4ac" (OK), "$\\\\Delta$" (FORBIDDEN).

4. **Logic Flow**:
   - First, select parameters in 'reasoning' based on difficulty.
   - Second, verify them mentally (ensure integer solutions or simple fractions).
   - Third, generate the question content using verified parameters.

# Output Schema Example
You MUST follow this structure exactly:
${exampleJson}`;
};

export const buildUserPrompt = (context = {}) => {
  const {
    motifName = '未知母题',
    specName = '通用专项',
    varName = '常规变式',
    difficultyConfig = { level: 'L2', tier: '基础筑基', complexity: 1, steps: 2, traps: 0, paramConstraint: 'integer' },
    variableKnobs = [],
    benchmarkQuestion = null
  } = context;

  const knobsDescription = variableKnobs.length > 0
    ? variableKnobs.map(k => `- ${k.name} (权重: ${k.weight || 1}): ${JSON.stringify(k.values || {})}`).join('\n')
    : '无特定参数池，请根据难度约束自行设计参数。';

  const benchmarkText = benchmarkQuestion?.problem || '无特定标杆，请自由发挥';

  return `## Context
- **母题主题**: ${motifName}
- **专项知识点**: ${specName}
- **变式方向**: ${varName}
- **目标难度**: ${difficultyConfig.level} (${difficultyConfig.tier})
  - 复杂度: ${difficultyConfig.complexity}
  - 步骤数: ${difficultyConfig.steps}
  - 陷阱数: ${difficultyConfig.traps}
  - 参数约束: ${difficultyConfig.paramConstraint}

## Reference Benchmark
参考标杆题逻辑（不要照抄，仅参考逻辑结构）:
${benchmarkText}

## Variable Knobs (Parameter Pool)
请从以下参数组合中，根据难度权重随机选择或调整一组作为基础参数：
${knobsDescription}

## Instructions
1. **Parameter Selection**: In 'reasoning', explicitly state which parameters you chose from the knobs (or adjusted) and WHY they fit the target difficulty.
2. **Self-Verification**: Ensure the chosen parameters lead to valid mathematical scenarios (e.g., discriminant >= 0, domain > 0).
3. **Question Generation**: Create a 2-part question (L1: Basic, L2: Advanced/Discussion) based on these parameters.
4. **Analysis**: Provide step-by-step solution with clear LaTeX formatting.
5. **Output**: Generate the JSON object immediately.`;
};
```
