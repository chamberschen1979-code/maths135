# 配置架构调整计划：M 文件驱动的策略选择

## 核心结论：建议完全正确，必须废弃旧的 knobsConfig.js

---

## 一、问题诊断

### M 文件 variable_knobs 的真实含义

通过分析 M06/M07/M08 文件，确认 variable_knobs 是**命题策略权重分布**，而非数值池：

```json
// M08 数列的 variable_knobs 示例
"variable_knobs": {
  "property_type": [
    { "type": "index_sum", "weight": 0.5, "desc": "下标和性质 ($a_m+a_n$)" },
    { "type": "segment_sum", "weight": 0.3, "difficulty_delta": "+0.5", "desc": "片段和成数列" }
  ],
  "trap_condition": [
    { "type": "q_sign_ambiguity", "weight": 0.6, "desc": "等比公比符号多解" }
  ]
}
```

**语义化 Type**：`index_sum`, `q_sign_ambiguity` 是数学逻辑标签
**领域专属**：三角函数关心 `angle_relation`，数列关心 `property_type`
**动态权重**：`weight` 控制 AI 选择某种考法的概率

### 旧 knobsConfig.js 的根本错误

```javascript
// 错误设计：定义具体数值
{ id: 'basic_int', values: { a: 2, b: 1, c: 3 } }
```

**冲突点**：
1. AI 拿到 `type: "index_sum"` 策略指令，却去 knobsConfig.js 找数值 → 逻辑断裂
2. M 文件精心设计的陷阱权重被通用随机数覆盖 → 考查意图偏移

## 二、新架构方案

### 数据流设计

```
┌─────────────────────────────────────────────────────────────┐
│  M 文件 (variable_knobs)                                    │
│  定义策略权重：property_type, trap_condition, calculation   │
└─────────────────────┬───────────────────────────────────────┘
                      ▼
┌─────────────────────────────────────────────────────────────┐
│  problemLogic.js (解析层)                                   │
│  1. 读取 motifData.specialties[].variations[].variable_knobs│
│  2. 根据 difficultyConfig 过滤过难选项                       │
│  3. 加权随机选择策略 → { property: {...}, trap: {...} }     │
└─────────────────────┬───────────────────────────────────────┘
                      ▼
┌─────────────────────────────────────────────────────────────┐
│  promptBuilder.js (Prompt 构建层)                           │
│  将策略转化为自然语言指令：                                   │
│  "核心考点：片段和成数列；必设陷阱：公比符号多解"              │
└─────────────────────┬───────────────────────────────────────┘
                      ▼
┌─────────────────────────────────────────────────────────────┐
│  AI (数值生成层)                                            │
│  根据策略自行生成具体数值（如 S_2=3, S_4=15）                │
└─────────────────────────────────────────────────────────────┘
```

### 步骤 3：重写 problemLogic.js 中的 selectVariableKnobs

```javascript
export const selectVariableKnobs = (motifData, targetLevel) => {
  const variation = motifData?.specialties?.[0]?.variations?.[0]
  
  if (!variation?.variable_knobs) {
    return { fallback: true, message: "使用通用策略" }
  }

  const knobs = variation.variable_knobs
  const selectedStrategy = {}

  for (const [dimension, options] of Object.entries(knobs)) {
    // 根据难度过滤
    const validOptions = options.filter(opt => {
      if (targetLevel === 'L2') return !opt.difficulty_delta || opt.difficulty_delta <= 0
      if (targetLevel === 'L3') return !opt.difficulty_delta || opt.difficulty_delta <= 0.5
      return true
    })

    if (validOptions.length === 0) continue

    // 加权随机选择
    const totalWeight = validOptions.reduce((sum, opt) => sum + (opt.weight || 1), 0)
    let random = Math.random() * totalWeight

    for (const opt of validOptions) {
      if (random <= (opt.weight || 1)) {
        selectedStrategy[dimension] = opt
        break
      }
      random -= (opt.weight || 1)
    }
  }

  return selectedStrategy
}
```

### 步骤 4：更新 promptBuilder.js

```javascript
export const buildUserPrompt = (context) => {
  const { variableKnobs } = context
  
  let strategyInstructions = []
  
  if (variableKnobs?.property_type) {
    strategyInstructions.push(`核心考点：${variableKnobs.property_type.desc}`)
  }
  if (variableKnobs?.trap_condition) {
    strategyInstructions.push(`必设陷阱：${variableKnobs.trap_condition.desc}`)
  }
  if (variableKnobs?.calculation_mode) {
    strategyInstructions.push(`计算要求：${variableKnobs.calculation_mode.desc}`)
  }

  return `
### 3. 命题策略约束
请严格按照以下策略生成题目：
${strategyInstructions.join('\n    ')}

注意：具体的数值请由你根据上述策略自行构造，确保数据整洁且可解。
`
}
```

| 风险 | 影响 | 缓解措施 |

## 七、结论

**采纳建议**，核心改动：
1. 废弃 knobsConfig.js（数值池）
2. 重写 selectVariableKnobs（策略选择）
3. 更新 promptBuilder（策略转 Prompt）
4. 让 AI 根据策略自行生成数值
