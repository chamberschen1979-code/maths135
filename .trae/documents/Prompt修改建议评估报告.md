# Prompt 修改建议评估报告

## 建议概述

将现有的 XML 格式输出改为 JSON Schema 格式输出，并大幅精简 Prompt。

---

## 评估分析

### ✅ 优点

| 优点 | 说明 |
|------|------|
| Token 大幅减少 | System Prompt 从 ~2000 Token 减少到 ~500 Token |
| 格式更明确 | JSON Schema 明确定义输出结构，减少格式错误 |
| 解析更简单 | JSON 解析比 XML 解析更可靠 |
| 结构化输出 | 便于前端渲染和数据处理 |
| 流式处理清晰 | 累积后一次性解析，避免中间状态问题 |

### ⚠️ 潜在问题

| 问题 | 影响 | 解决方案 |
|------|------|----------|
| 输出格式变更 | 现有 XML 解析逻辑需要替换 | 需要修改 `cleanAIContent` 和解析逻辑 |
| 前端渲染适配 | 现有组件依赖 XML 格式 | 需要修改渲染组件 |
| 约束减少 | 可能影响出题质量 | 需要测试验证 |
| LaTeX 格式 | JSON 中 LaTeX 可能被转义 | 需要处理转义字符 |

### ❌ 需要保留的内容

| 内容 | 原因 |
|------|------|
| 难度分级宪法 | 确保题目难度符合标准 |
| 参考标杆 (Few-Shot) | 提供示例，提高生成质量 |
| 变量旋钮 | 确保题目多样性 |
| 解析绑定铁律 | 确保解析与题目匹配 |

---

## 对比分析

### 现有方案 vs 建议方案

| 维度 | 现有方案 | 建议方案 |
|------|----------|----------|
| 输出格式 | XML | JSON |
| Prompt 长度 | ~3500 Token | ~1000 Token |
| 解析复杂度 | 高（XML + JSON 兼容） | 低（纯 JSON） |
| 出题质量 | 高（约束详细） | 待验证（约束精简） |
| 生成速度 | 慢 | 快 |

---

## 建议修改

### 保留关键约束

建议在精简的 System Prompt 中保留以下关键约束：

```javascript
const systemPrompt = `
# Role
高中数学命题专家。基于母题生成符合新高考 I 卷标准的原创题目。

# Constraints
1. **Format**: 仅输出纯 JSON 对象。严禁 Markdown 包裹。
2. **LaTeX**: 所有公式用 $...$ 包裹。中文严禁放入 $ 内。
3. **Logic**:
   - Q1: 基础应用，路径清晰，参数简单。
   - Q2: 逻辑延伸，需分类讨论或构造，参数稍复杂但可手算。
   - 判别式Δ需为完全平方数 (若适用)。
4. **Content**: 解析必须代入具体参数值，禁止通用模板。
5. **Quality**: 保持学科基因耦合（如复数题必含 i），陷阱需符合高考习惯。

# Output Schema
{
  "reasoning": {
    "q1_params": "选定的参数及理由",
    "q2_params": "选定的参数及理由",
    "design_logic": "简要设计思路"
  },
  "question": {
    "difficulty_l1": "基础巩固|综合运用|拓展探究",
    "difficulty_l2": "基础巩固|综合运用|拓展探究",
    "content": "题干内容，含 (1)(2) 问，公式用 LaTeX"
  },
  "analysis": {
    "core_idea": "一句话核心思路",
    "steps": ["步骤1...", "步骤2...", "..."],
    "trap_hint": "易错点提示"
  },
  "answer": {
    "l1": "答案1",
    "l2": "答案2"
  }
}
`;
```

### 需要修改的代码

1. **`generateAIProblem` 函数**：替换 systemPrompt 和 userPrompt
2. **解析逻辑**：移除 XML 解析，使用纯 JSON 解析
3. **前端渲染**：适配新的数据结构

## 结论

### 评估结果：**建议采纳，但需要谨慎实施**

### 实施步骤

1. **第一步**：修改 `generateAIProblem` 函数中的 Prompt
2. **第二步**：修改解析逻辑，使用 JSON 解析
3. **第三步**：修改前端渲染组件
4. **第四步**：测试验证出题质量

### 风险提示

- 出题质量可能下降，需要测试验证
- 前端渲染需要适配新数据结构
- 建议先在测试环境验证后再上线

---

## 是否采纳建议？

**我的建议是：可以采纳，但需要：**
1. 保留关键约束（难度分级、学科基因、解析绑定）
2. 逐步实施，先测试再上线
3. 准备回滚方案，以防出题质量下降
