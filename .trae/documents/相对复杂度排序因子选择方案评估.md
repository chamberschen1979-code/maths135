# 基于相对复杂度排序的因子选择方案评估

## 建议概述

将硬编码的数值特征描述（"整数/根号"）改为"相对复杂度排序"指令：
- L2：从因子池中挑选【复杂度最低】的部分
- L3：从因子池中挑选【中等复杂度】的部分
- L4：从因子池中挑选【复杂度最高】的部分

---

## 核心优势分析

### 1. 去除数值偏见 ✅

**旧方案问题**：
```
L2 = 整数
L4 = 根号/无理数
```
这在代数题（M09）管用，但在：
- 概率题（M05）：难点是"分类讨论层数"或"条件概率嵌套"
- 立体几何（M07）：难点是"非特殊角"或"动点轨迹"

**新方案优势**：
```
L2 = 复杂度最低的因子
L4 = 复杂度最高的因子
```
AI根据数学直觉自动判断：
- 概率题：P(A)=0.5 < P(A|B∩C)
- 立几题：中点 < 三等分点，垂直 < 任意角
- 代数题：整数 < 根号

### 2. 完全复用现有数据结构 ✅

**不需要修改 JSON 结构**：
```json
{
  "variable_knobs": {
    "element_type": [
      { "type": "number_set", "weight": 0.6 },
      { "type": "point_set", "weight": 0.3, "difficulty_delta": "+0.5" },
      { "type": "abstract_set", "weight": 0.1, "difficulty_delta": "+1" }
    ]
  }
}
```

AI可以直接从 `weight` 和 `difficulty_delta` 感知复杂度排序。

### 3. 利用AI的数学直觉 ✅

AI天然具备对数学对象复杂度的判断力：
- 不需要我们告诉它"什么是难"
- 只需要告诉它"怎么选"（取头还是取尾）

---

## 实施方案

### Prompt 模板

```
核心约束：双难度分层策略 (泛用型)

第 (1) 问 (基础层)
难度目标: 【L2】
取值指令: 请分析【可用变量因子池】，从因子池中挑选【复杂度最低】的那一部分数值/参数用于本问。
注意：不要编造池中不存在的参数类型。如果是概率题，简单可能意味着"事件少、条件直接"；如果是立体几何，简单可能意味着"特殊位置、特殊角"。请依据数学直觉判断复杂度。

第 (2) 问 (提升层)
难度目标: 【L4】
取值指令: 请再次分析【可用变量因子池】，从因子池中挑选【复杂度最高】的那一部分数值/参数用于本问。
注意：确保本问选用的参数组合在逻辑深度或计算复杂度上，明显高于或不同于第 (1) 问。
```

### 代码实现

```javascript
const getSelectionStrategy = (level) => {
  switch(level) {
    case 'L2': return "从因子池中挑选【复杂度最低】的那一部分数值/参数（最基础、最直观的配置）";
    case 'L3': return "从因子池中挑选【中等复杂度】的数值/参数（具有一定计算量或逻辑转折）";
    case 'L4': return "从因子池中挑选【复杂度最高】的那一部分数值/参数（最繁琐、最隐蔽或最具综合性的配置）";
    default: return "从因子池中随机挑选";
  }
};
```
